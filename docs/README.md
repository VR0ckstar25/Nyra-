# Nyara Documentation Map

Use this folder as the implementation map. The root app and database folders contain
working code/data; this folder contains the product, security, legal, and design
source documents.

## Build-order source

1. `00_source_of_truth/Anvara_DevExecSequence_v1.docx`
2. `00_source_of_truth/Anvara_Decisions_Log.md`
3. `01_product_flow/Anvara_Amendments_v3_1_TransferNote.docx`
4. `01_product_flow/Anvara_1_ClaudeCode_TransferNote_v3.docx`
5. `04_design/Anvara_7_ClaudeDesign_TransferNote_v1.docx`

When documents disagree, follow the order above. The decisions log exists to reconcile
late founder decisions, security corrections, and pricing/family-plan changes.

## Folder guide

- `00_source_of_truth/` — active build sequence and reconciled decisions.
- `01_product_flow/` — product-flow transfer notes and amendments.
- `02_security_privacy_legal/` — legal/regulatory guardrails and privacy posture.
- `04_design/` — design note, PDF, and UI handoff package.
- `05_regulatory_references/` — FDA and related reference PDFs.
- `99_archive_raw_exports/` — reserved for quarantine notes only; stale raw exports
  with superseded guidance were removed.

Do not restore or build from raw exports that contain superseded wording.
